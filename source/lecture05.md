### 繰返し回数を「整数値」で与えるための工夫

`for i in range(10):` のような構文で**繰返し処理**を実行する場合、`range` に与える引数 (ひきすう) を <span class="masked">整数</span> にしないと**実行時エラーが発生します**。以下のコードで実際に確認してください。

- 次のプログラムでは `range` の引数 (=繰り返し回数) が「整数」ではなく「小数」になってしまっています。

```python{.numberLines caption="NG例（繰返し回数が小数値になる例）"}
%reset -f
n=7
for i in range(360/n): # 360/7=51.42.....
  print('AA')
```

実際に実行すると <span class="masked">Type Error（型の誤り）</span> が発生します。

図形処理では `range` の引数を「数式」で与えたい場合がありますが (例えば`360/n` のようなケース)、このような場合、実行時エラーを回避するためには次のような工夫をします。

```python{.numberLines caption="繰返し回数を整数指定 (1)"}
%reset -f
n=7
for i in range(360//n): # / の代わりに // を使用
  print('AA')
```

```python{.numberLines caption="繰返し回数を整数指定 (2)"}
%reset -f
n=7
for i in range(int(360/n)): # int() を使用
  print('AA')
```

また、目的に応じて `range(360//n+1)` のように繰返し回数を調整します。慣れないうちは、適宜、括弧を使って `range((360//n)+1)` のように記述すると分かりやすいかもしれません。

- `//` 演算子は [第03回講義](lecture03.html#四則演算) で既に学習済みです。
- `int` 関数は [第04回講義](lecture04.html#文字列を整数値に変換する-int関数) で既に学習済みです。
  - `int` 関数は「文字列」を「整数値」にするほか、「実数値」を「整数値」にするときにも利用できます。
  - `int` 関数で、「実数値」を「整数値」に変換するとき、四捨五入、切り上げ、切り捨てのいずれになるかは実際に自分で確認してみてください。`print(int(4.5))`

## 条件分岐構文

***条件分岐構文*** (通称「**if文**」) は、何らかの条件に応じて処理の流れを変えたい (つまり分岐させたい) ときに使用します。具体的には `num <= 4 ` のような「条件式」を与えて、それが

- **成立するか** (**真**であるか)
- **成立しないか** (**偽**であるか)

によって実行するコードを変えます。

繰返し構文と同様に <span class="masked">処理の範囲 (=ブロック) をインデントにより指示</span> します。

実例で考えてみましょう。次のプログラムは、サイコロ🎲を投げて「１～4の目がでたとき」は「**はずれ**」、一方で「5または6がでたとき」は「**あたり**」の処理をするものです。実際に実行して動作を確認してください。

```python{.numberLines caption="サイコロで運試しゲーム.py"}
%reset -f
import random

for i in range(5): # 5回チャレンジ

  dice_num = random.randint(1,6) # [1,6]の範囲の乱数生成
  print(f'サイコロで{dice_num}がでたので「',end='')

  # 条件分岐構文 ⚠️⚠️ 条件式を括弧で囲まないことに注意 ⚠️⚠️

  if dice_num <= 4 :  # 末尾のコロン (:) を忘れがちなので注意
    print('はずれ',end='')

  else :             # 末尾のコロン (:) を忘れがちなので注意
    print('あたり',end='')

  print('」です。')
```

なお、Python 以外の言語、例えば C/C++言語 (Arduino言語を含む) などでは、<span class="masked">条件式を `(` と `)` で囲みますが、Python言語では囲まない</span> ので注意してください。

### アタマを柔らかくして分岐条件を与える②

<u>条件を満たすときは**何もせず**、条件を**満たさないときだけ何かをしたい**</u>という処理はどのように記述すればよいか考えてみてください。

例えば、

>サイコロを投げて6の目がでたときは**何もせず**、それ以外のときは「**はずれ**」を表示する

という処理を考えます。まず、はじめに思い浮かぶのは、次のようなプログラムになると思います。**実際に実行して結果を確認**してください。

```python{.numberLines caption="問題のある実装"}
%reset -f
import random
dice_num = random.randint(1,6)
if dice_num == 6:
  # 何も処理しない
else:
  print('はずれ')
```

実行すると <span class="masked">エラーとなることが確認できたと思います (実行以前の段階で赤の波線が表示されていることにも気づきましたか?) </span>。

Pythonでは、文法上、**何も処理しないブロック (文が１つも存在しないブロック) を構成することは禁止されています**。ブロック内には、(コメント以外に) 必ず何らかの文を書く必要があります。そこで、何もしない文として `pass` が存在します。

上記のプログラムは次のように書き換えることで実行可能となります。

```python{.numberLines caption="pass文を記述して問題を回避した実装"}
%reset -f
import random
dice_num = random.randint(1,6)
if dice_num == 6:
  pass # 何も処理しない。
else:
  print('はずれ')
```

しかし、`pass` を使用しなくとも、次のように書き換えることで、より短いコードで同じ処理を実現することができます。

```python{.numberLines caption="賢い実装 (可読性も向上)"}
%reset -f
import random
dice_num = random.randint(1,6)
if dice_num != 6:
  print('はずれ')
```

### elif節

if score >= 90 :

### 条件分岐構文のネスト

***ネスト*** とは **入れ子** とも呼ばれるもので、条件分岐構文のなかに、さらに条件分岐構文を構成することを意味します。


基本的には何階層にも入れ子にすることができますが、深いネストを構築すると <span class="masked">可読性や保守性が著しく低下</span> します。深いネストが構成されないようにプログラムを工夫してください。

```python{.numberLines caption="条件分岐のネスト（入れ子）"}
# @title 点Aは第何象限に存在？ { run: "auto" }

Ay = -5  #@param {type:"slider", min:-5, max:5, step:1}

  if Ax > 0 :
    print('1',end='')

    print('2',end='')

  if Ax < 0 :
    print('3',end='')
  else :

```

### AND条件

***AND条件*** は `and` 演算子を使って評価することができます。C、Java、JavaScript など多くの言語では `&&` が使われますが、Python では `&&` は使用できず、構文エラーになるので注意してください。

```python{.numberLines caption="AND条件を使用.py"}
%reset -f
x = 0  # 点AのX座標
y = 0  # 点AのY座標
if x==0 and y==0 :  #「and演算子」を利用
  print('点Aは原点上に存在します')
else :
  print('点Aは原点以外に存在します')
```

上記と同じ処理をするプログラムは、次のように **条件分岐構文のネストでも構成** することもできます。ただし、可読性・保守性に問題があることを実感してください。

```python{.numberLines caption="AND条件を使用せずに同じ処理するプログラム.py"}
%reset -f
x = 0
y = 0
if x==0 :
  if y==0 :
    print('点Aは原点上に存在します')
else :
  print('点Aは原点以外に存在します')
```

`and` 演算子では、`x==0 and y==0 and z==0` のように **3つ以上の条件** についての **AND (論理積)** をとることもできます。

### 条件式の正体

`x > 5` や `x==0`、`0<=x<=10` のような条件式は、次のように <span class="masked">if文以外の場所</span> でも使うことができます。


```python{.numberLines caption="条件式は「True」か「False」を返す計算式"}
%reset -f
x = 10

# x > 5 の評価結果を 変数var1 に格納
var1 = x > 5  

# x == 0 の評価結果を 変数var2 に格納
var2 = x == 0  

# 変数に格納された値を出力
print(f'var1={var1}, var2={var2}')
```

実行結果は次のようになります (面倒がらずに実際に実行してください)。

```
var1=True, var2=False
```

上記の実行結果から分かるように、比較演算子を使った条件式は、`True` または `False` という <span class="masked">bool型 (ブール型、ブーリアン型、真偽値型) の値</span> になります。

**bool型** は、`True` または `False` の2値から構成される値です。int型 (整数型) や float型 (浮動小数点数型) と同様に変数に格納 (バインド) することができます。

そのため、次のように利用することもできます。

```python{.numberLines caption="条件の評価結果を変数に格納してから条件判定を行う例"}
%reset -f
x = -5
y = 12
z = 80
c1 = -10 <= x <= 10 # 条件式の評価結果をc1に格納
c2 = -15 <= y <= 15 #  〃 をc2に格納
c3 = -20 <= z <= 20 #  〃 をc3に格納
print('点Aは指定の空間内に',end='')
if c1 and c2 and c3 :
  print('存在します。')
else :
  print('存在しません。')
```

変数に格納せずに、`if` に直接的に条件を与えると次のようになります。

```python{.numberLines caption="条件をif文に直接記述"}
%reset -f
x = -5
y = 12
z = 80
print('点Aは指定の空間内に',end='')
if -10 <= x <= 10 and -15 <= y <= 15 and -20 <= z <= 20 :
  print('存在します。')
else :
  print('存在しません。')
```
